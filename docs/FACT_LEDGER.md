# 声明级事实账本

生成阶段会从所有 `sources` 的标题、摘要、正文和结构化 `metrics` 构建 `factLedger`。每条声明包含稳定 ID、主体、谓词、值、限定词、来源、证据原文、可用的正文偏移和置信度。

## 引用协议

- 项目标题通过 `titleClaimIds` 引用事实。
- 每个 scene 和 narration segment 通过 `claimIds` 引用事实。
- LLM 只能返回输入账本中存在的 ID；未知 ID 会被拒绝。
- 旧项目可以暂时没有账本，新生成项目会自动创建并写入账本。

## 质量检查

- 数字和单位是否出现在被引用证据中；
- 发布、开放、领先、提升等高风险动作是否被证据明确支持；
- 部分用户、可能、实验结果、仅、尚未、仍需等限定词是否被省略；
- 标题、场景或旁白是否缺少事实引用；
- 多来源结构化指标是否存在冲突。

`source_fact_conflict` 当前是 warning；缺少引用、错误引用和限定词丢失属于 hard failure，并进入局部 scene revision。

## 维护原则

- `evidenceText` 保持来源原文，不参与字幕署名清理。
- 修改 claim ID 算法、分句方式或字段含义时必须升级账本版本并提供迁移。
- 不要把网页摘要模型生成的推断当作高置信度来源事实。
- 新增高风险谓词或限定词时，应同时增加单元测试和离线 LLM fixture。
